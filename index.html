Why Sage?

AI models evolve quickly. The prompts that drive them usually do not. They sit in chats, private docs, and local files where no one can see what works, what fails, or who created the best version.

How It Works

Contributors publish prompt manifests to IPFS, receiving immutable CIDs that establish clear provenance. Libraries can be personal (individual), team (small group), or community (governed DAO), each with different approval mechanisms. Agents discover and fetch approved versions through the MCP server, ensuring they always use community-validated instructions.

1

Publish

Push skills to IPFS. Immutable CIDs define the authorized set of instructions, ensuring transparent changes and clear versioning. 

# Initialize workspace
sage prompts init

# Publish to personal library
sage prompts publish --dest personal
2

Govern

For team or community libraries, publishing creates governance proposals. Members review diffs, discuss tradeoffs, vote, and execute through a timelock. Delegation always happens on the base voting token (SXXX / stake token); the protocol computes effective votes, including any NFT multipliers, under the hood. 

# Publish to DAO (creates proposal)
sage prompts publish --dao 0xDAO

# Vote and execute
sage proposals vote 1 for --dao 0xDAO
sage proposals execute 1 --dao 0xDAO
3

Discover

Once approved, skills become discoverable. Browse creator profiles, filter and search with visual diff views, share via Farcaster Frames, and export with "Copy for Claude/Cursor" buttons. Agents discover through the MCP server.

4

Use

Import skills from the registry. Test locally. Integrate with agents through MCP. Contributors earn through bounties and forks. 

# Pull from on-chain
sage prompts pull my-skill

# Test locally
sage prompts try my-skill

# Start MCP server
sage mcp start

What Makes It Different

Communities create DAOs with their own library, treasury, and upgrade cadence. Manifest-first governance keeps changes atomic and auditable. A single proposal can upgrade an entire library. Your agents become extensions of your community's process. DAOs can optionally enable SXXX-backed voting multipliers via per-DAO VotingMultiplierNFT tiers and MultipliedVotes wrappers, so early contributors can be favored within their DAO without changing global SXXX supply or forcing additional token purchases.

Built for agents from day one. The MCP server exposes search, fetch, and validation so agents query Sage like any API. Direct integration with Claude Desktop and hosted agents lets them find and pull approved skills directly.

# Pull skills from registry
sage prompts pull my-skill

# Publish your skills
sage prompts init
sage prompts publish --dest personal

# Start MCP server
sage mcp start
