Why Sage?

AI models evolve quickly. The prompts that drive them usually do not. They sit in chats, private docs, and local files where no one can see what works, what fails, or who created the best version.

Sage exists so communities can govern, discover, and reward the prompts that actually move the needle.

Prompts as Public Goods

Sage treats prompts as shared infrastructure. Governance participants capture value through their own use of prompts that are aligned with their needs. Approved prompts are cultivated as public goods and kept in versioned libraries with clear provenance.

Communities can fork, adapt, and reuse these skills while preserving attribution and trust. Your best instructions stop living in a single notebook and start living as a governed resource for your entire network.

A Network that Reveals High Value Prompts

High value prompts are hard to identify in isolation. They maximize subjective value per token per user and that signal only really emerges at network scale. Sage uses incentives and transparent usage to surface what works.

When a prompt is used, value moves and the network records that interaction. Libraries become an archive of outcomes. Success and failure both compound so teams can study both and improve faster.

Permissionless Collaboration with Provenance

Sage is built for permissionless collaboration around AI skills, including Claude and other agent platforms. Anyone can propose and refine new skills. Communities approve, adopt, and standardize them for agents to use. On-chain attribution records which wallets deliver high value skills.

Value accrues to the network and to the aligned communities that cultivate it. Contributors keep credit and upside for the intelligence they help create.

How It Works

A simple path from a single prompt idea to a governed, discoverable agent skill.

1

Publish

Push manifests and prompt files to IPFS. Immutable CIDs define the authorized set of instructions for a given skill or library, ensuring transparent changes and clear versioning. 

# Connect your wallet
sage wallet connect --type cast
sage doctor

# Publish a library
sage library push manifest.json --pin
2

Propose

Each manifest becomes one governance proposal in your SubDAO. Members can review diffs and context, discuss tradeoffs, vote under your chosen rules, and queue and execute through a timelock.

Upgrades happen in the open and are always tied to a concrete manifest.

3

Discover

Once approved, the Prompt Registry marks the new CID as canonical for that library. Agents, the CLI, and the web app read via subgraph first, fall back to RPC when needed, and always resolve to a consistent view of the approved instructions.

Sage acts as a discovery engine for agent capabilities, so new skills and improvements are always easy to find and safe to reuse.

4

Use

MCP tools return content by CID so agents can orchestrate tasks with the community-approved version. 

# Start HTTP MCP server for AI agents
sage mcp start

# Or run stdio server for Claude Desktop
node packages/cli/src/mcp-server-stdio.js

Every interaction feeds back into the system through value exchange and recorded usage. The network learns from its own consumption and turns context into an economic and informational flywheel.
