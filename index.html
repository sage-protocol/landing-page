Why Sage?

AI models evolve quickly. The prompts that drive them usually do not. They sit in chats, private docs, and local files where no one can see what works, what fails, or who created the best version.

How It Works

A simple path from a single prompt idea to a governed, discoverable agent skill.

1

Publish

Push manifests and prompt files to IPFS. Immutable CIDs define the authorized set of instructions, ensuring transparent changes and clear versioning. 

# Connect your wallet
sage wallet connect --type cast
sage doctor

# Publish a library
sage library push manifest.json --pin
2

Propose

Each manifest becomes one governance proposal in your SubDAO. Members review diffs, discuss tradeoffs, vote, and execute through a timelock. Upgrades happen in the open and are tied to a concrete manifest.

3

Discover

Once approved, the Prompt Registry marks the new CID as canonical. Agents, the CLI, and the web app read via subgraph first, fall back to RPC when needed, and resolve to a consistent view. Sage acts as a discovery engine for agent capabilities.

4

Use

MCP tools return content by CID so agents orchestrate tasks with the community-approved version. Every interaction feeds back through value exchange and recorded usage. The network learns and turns context into an economic and informational flywheel. 

# Start HTTP MCP server for AI agents
sage mcp start

# Or run stdio server for Claude Desktop
node packages/cli/src/mcp-server-stdio.js
